## ADR-004: Server-Side Credential Injection

Downstream credentials are issued and injected server-side instead of handed to the agent as raw API keys or refresh tokens. This reduces the blast radius of prompt injection, accidental logging, memory scraping, and agent-side exfiltration. The runtime receives an authorization outcome, not durable secret material.

The alternative would be to let the agent fetch and hold external credentials directly after session approval. That simplifies some runtime integration code but defeats one of the project's primary security goals. Once a runtime can persist or reuse raw secrets, the identity plane loses meaningful control over how those secrets are used.

By separating session approval from credential issuance, the system can rotate providers, narrow leases, and keep a durable record of which downstream credential lease was created for which session without exposing long-lived secrets to the workload.

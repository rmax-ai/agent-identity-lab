## ADR-007: MCP Gateway Enforcement Boundary

The MCP gateway is treated as the mandatory enforcement boundary between agents and MCP tools. That means the gateway is responsible for validating session context, obtaining authorization decisions, and ensuring downstream credential use stays inside the platform's control plane instead of being delegated to the agent runtime.

Without this boundary, agents could call tools directly once they obtained some form of credential or endpoint knowledge. That would fragment policy enforcement and make audit trails incomplete. Centralizing the check in the gateway preserves one place where access rules, session state, and tool invocation can be correlated.

This ADR is what turns the rest of the system from a credential-issuing library into an actual control plane. Blueprints, sessions, and policy only matter if tool execution is forced through an enforcement point that honors them.

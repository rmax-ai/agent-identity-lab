## ADR-001: Agents As First-Class Principals

The platform models each agent as its own principal rather than collapsing agent actions into a generic service account. This decision preserves attribution at the right boundary: the system can answer which specific agent acted, which blueprint authorized it, and which runtime instance presented the attestation for that session.

This increases modeling complexity because the system must manage agent lifecycle state, key material, and session issuance directly. The tradeoff is worth it because it aligns security controls with how autonomous runtimes actually behave. An agent can be suspended, rotated, or reviewed independently of the human user or the hosting service.

The result is a clearer audit model and a more precise authorization surface. Instead of reasoning about "the app" as one actor, operators can reason about each deployed agent identity as an accountable workload.

## ADR-006: Tamper-Evident Audit Chain

Audit events are linked through a hash chain so later reviewers can detect modification or removal. The goal is not just to log actions, but to make those logs meaningful in a post-incident review where integrity matters. If the audit trail can be edited silently, it cannot serve as a trustworthy accountability layer.

This is more complex than plain append-only logging because each event now depends on prior chain state and verification logic. That extra rigor is justified because the platform is explicitly about agent accountability: who acted, under what policy, and with what credential path.

The ADR does not claim that a local hash chain solves every audit-integrity problem. It does establish a stronger default than ordinary application logs and gives the system a concrete place to anchor future external attestation or log shipping.

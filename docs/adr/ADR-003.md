## ADR-003: Permission Intersection Model

The authorization model computes effective permissions by intersection rather than union. Requested scopes must fit inside the blueprint contract, the agent's current posture, the tool operation's required scopes, and, when present, the acting user's authority. This prevents a single permissive layer from silently broadening access.

The cost of this approach is that it can feel strict and sometimes produce denials that require debugging across multiple inputs. That is acceptable because false allows are more dangerous here than false denies. A denied action can be investigated; an over-broad allowed action can leak data or mutate external systems.

This ADR is central to the lab's identity story. It turns agent permissions into an explicit contract that can be audited and reasoned about instead of a best-effort convention at runtime.
